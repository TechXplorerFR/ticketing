# ADR - Choix Framework Frontend 

## Contexte

On cherche ici à définir le framework utilisé pour le frontend.

## Décision

Suite à une veille technologique, notre choix s'est porté sur `Node.js`, `React.ts`, `Vite` et `tailwindcss`.

Les raisons de ce choix sont les suivantes :

_Node.js_ :

- Très répandu
- Facilement modifiable
- Multitude de module

_React.ts_ :

- Fréquemment utilisé pour des sites simples et interactifs
- Facilement testable

_Vite_ :

- Interface de développement simplifiée pour le développement
- Fournit des plugins de tests complets
- Grande communauté pour le support

_Tailwindcss_ :

- Très utilisé
- Facile de compréhension
- Classe couvrant les principaux cas d'usages du CSS

## Statut

✅ - Validé à l'unanimité

## Conséquences

Le frontend sera structuré et simple à tester et à développer pour intégrer de nouvelles fonctionnalités
